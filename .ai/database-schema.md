# QuickTools.ai - Database Schema

## Overview
Using a relational database like **PostgreSQL** (via Supabase, Neon, or Vercel Postgres).
ORM: **Prisma** or **Drizzle**.

## Core Tables

### 1. `User`
- `id` (UUID, Primary Key)
- `email` (String, Unique)
- `name` (String, Optional)
- `image` (String, Optional)
- `plan` (Enum: FREE, PRO) - Default: FREE
- `credits` (Int) - Tracks AI tool usage limits for free tier
- `createdAt`, `updatedAt` (DateTime)

### 2. `BlogPost` (For Auto-Blogging System)
- `id` (UUID, Primary Key)
- `slug` (String, Unique) - e.g., `best-ai-tools-2026` (For SEO)
- `title` (String)
- `content` (Text/Markdown) - The actual AI-generated blog content
- `metaDescription` (String) - Generated by AI specifically for Google search
- `keywords` (String[]) - Array of tags/keywords
- `status` (Enum: DRAFT, PUBLISHED) - Default: PUBLISHED (Auto-magic)
- `publishedAt` (DateTime)
- `createdAt`, `updatedAt` (DateTime)

### 3. `ToolUsage` (Analytics & Tracking)
- `id` (UUID, Primary Key)
- `userId` (UUID, Foreign Key -> User.id) - Optional (if allowing guest usage)
- `toolName` (String) - E.g., 'text-summarizer'
- `usedAt` (DateTime) - Timestamp of when the tool was used
